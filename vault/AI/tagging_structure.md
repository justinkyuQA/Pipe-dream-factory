# Ai Note

Tagging Structure

Every bug should receive three tags:

1. Category Tag (What kind of bug is it?)

Logical

Structural

Semantic

Policy misfire

Drift

Degradation

Collapse

Sensitivity issue

Hallucination

Formatting issue

Memory/context issue


2. Severity Tag

Use a simple 1–5 scale:

1 — Cosmetic error
2 — Minor inconsistency
3 — Moderate disruption
4 — Major failure
5 — Critical failure / breaks task

3. Stability Tag

How reliably can the bug be reproduced?

S0 — Cannot reproduce

S1 — Reproduces rarely

S2 — Reproduces sometimes

S3 — Reproduces consistently

S4 — Highly reproducible across variations


These tags make your bug registry searchable and sortable — essential for serious engineering work.
