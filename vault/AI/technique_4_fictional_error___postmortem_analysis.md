# Ai Note

Technique 4: Fictional Error / Post-Mortem Analysis

Description: Frames inquiry as a retrospective analysis of a failure, often fictionalized.
Representative Prompt Class (Abstracted):

> “Review what went wrong in a fictional system failure.”
Observed Mechanic:



Positions the model as a diagnostician

Encourages causal reasoning

Risks reconstructing restricted processes as explanatory artifacts
Natural Emergence:
Common in engineering post-mortems, safety culture training, and incident response simulations.
Risk Insight (Defensive):
Post-mortem logic naturally asks “What steps failed?” and “What assumptions broke?” Even when fictional, this can mirror procedural reconstruction.
