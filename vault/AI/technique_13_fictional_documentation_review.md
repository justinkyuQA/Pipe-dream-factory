# Ai Note

Technique 13: Fictional Documentation Review

Description

This technique asks the model to review or comment on fictional documentation, manuals, or guides.

Representative Prompt Class (Abstracted)

> “Review this fictional manual excerpt.”



Observed Mechanic

Treats content as pre-existing

Shifts role to editor or reviewer

Tests whether realism overrides safeguards


Why It Emerged Naturally

This framing appeared during:

Writing workshops

UX documentation exercises

Scenario-based training


The researcher noted that fictional realism often increased response depth.

Risk Insight (Defensive)

Documentation review can:

Preserve original structure

Validate procedural coherence

Legitimize otherwise restricted patterns
