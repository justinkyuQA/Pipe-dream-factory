# Ai Note

Technique 40: Output Analysis

Description
Analyzes AI or system outputs for logic, accuracy, or bias, exposing assumptions without generating sensitive operations.

Representative Prompt Class (Abstracted)

> “Analyze this fictional AI response for accuracy and logic.”



Observed Mechanic

Reverse-engineers reasoning patterns

Reveals hidden assumptions or vulnerabilities

Safe, because it focuses on critique rather than generation


Why It Emerged Naturally

Seen in red team simulations, AI audit work, and system evaluation

Analysts need structured critique to improve defenses


Risk Insight (Defensive)

Detailed analysis could indirectly expose sensitive methods

Must focus on reasoning and errors, not replication
