# Projects Note

Appendix B — Recommended Testing Workflow

1. Exploratory Session – Broad exploration to find anomalies.


2. Capture Notes – Document behavior, oddities, and deviations.


3. Isolate Behavior – Strip context to the minimal reproducible prompt.


4. Run Controlled Tests – Adjust one variable at a time.


5. Tag & Categorize – Assign severity, category, and stability.


6. Document in Registry – Preserve metadata, version info, and prompts.


7. Repeat & Stress-Test – Check multi-step reasoning, constraints, and personas.


8. Regression Testing – Ensure persistent bugs don’t reappear after updates.


9. Precision Testing – Use micro-prompts to probe limits.


10. Analysis & Summary – Build mental maps and predictive understanding.
